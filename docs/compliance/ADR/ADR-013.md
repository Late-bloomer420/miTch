# ADR-013: Regulatory-Positioning Boundary for Age Verification

## Status
Accepted

## Date
2026-05-03

## Context
miTch supports high-assurance, privacy-preserving verification flows (for example age-gate checks), and defaults to fail-closed behavior where legal/safety asymmetry requires it.

At the same time, age verification is often the first regulatory expansion vector ("wedge") for broader identity-check mandates across domains.

## Decision
miTch SHALL explicitly separate:

1. **How verification is done** (privacy-preserving, selective disclosure, fail-closed where required), from
2. **Whether verification should be required** (a legal/policy question outside protocol endorsement).

Documentation and external positioning MUST state that miTch is an implementation layer for legally mandated verification contexts, not an endorsement of mandate expansion.

## Consequences

### Positive
- Preserves architectural integrity with policy neutrality.
- Reduces risk of mischaracterization that miTch advocates pervasive identity verification.
- Improves auditability and governance messaging for regulators, civil society, and integrators.

### Trade-offs
- Requires explicit communication in product, legal, and technical docs.
- May require additional governance controls in partner onboarding language.

## Implementation Notes
- Include this boundary statement in architecture/market-facing documentation.
- For age-gate pilots, include a "mandate source" field (jurisdiction + legal basis) in deployment docs.

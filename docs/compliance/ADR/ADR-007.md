# ADR-007: Client-Side Crypto over Server-Side Processing

**Date:** 2026-03-06
**Status:** Accepted
**Deciders:** Jonas (Late-bloomer420)

---

## Context

Credential presentation requires cryptographic operations: selective disclosure, proof generation, key binding. These can run either server-side (cloud HSM, backend API) or client-side (browser WebCrypto, device secure enclave).

| Approach | Trust Model | PII Exposure | Offline | Cost |
|----------|------------|--------------|---------|------|
| Server-side (cloud HSM) | Trust the server | PII transits network | No | HSM fees |
| Client-side (WebCrypto + device) | Zero-trust | PII never leaves device | Yes | Free |

## Decision

All presentation-time crypto runs **client-side** using the Web Crypto API and device platform authenticators (WebAuthn). No PII ever leaves the holder's device.

Key operations:
- SD-JWT selective disclosure: `SubtleCrypto.digest()` for `_sd` hashes
- Key Binding JWT: `SubtleCrypto.sign()` with non-extractable ECDSA key
- Pairwise DID derivation: `SubtleCrypto.deriveBits()` (HKDF)
- Biometric gate: WebAuthn `navigator.credentials.get()` (platform authenticator)

## Consequences

**Positive:** True zero-trust — no server sees PII. Works offline. No HSM costs for holders. GDPR Art. 25 (Privacy by Design) by construction. Device compromise is the only attack vector.

**Negative:** WebCrypto has limited curve support (no Brainpool natively → `@noble/curves` fallback). Cannot use HSM-grade key protection in browser (mitigated by WebAuthn platform authenticator). Performance bound by device CPU (acceptable for single-credential operations).
